The FAQ page is for frequently asked and general questions about OData. If you have more questions, please ask in the comments below. We will regularly collect questions from the comments (and other channels like StackOverflow, Twitter and mailing list) then put them on the FAQ page.

Q: OData versions and compatibility?

Currently, there are four versions of OData: OData version 1.0, 2.0, 3.0 and 4.0. OData was initiated by Microsoft in year 2007. Version 1.0, 2.0 and 3.0 are released under the Microsoft Open Specification Promise. Version 4.0 was standarlized at OASIS, with a release in March 2014. And in April 2015, OASIS has further submitted OData v4 and OData JSON Format v4 to ISO/IEC JTC 1 for approval as an International Standard.

Q: What is service document and metadata document?

Service document advertises the top-level resources which can be accessed on the service root. Metadata on the other hand represents the data model of the REST service. Service document is a MUST for every OData service and metadata document is RECOMMENDED. Metadata document is very important for clients to understand your service.

Q: Does OData support transactions?

Yes. Although the term "transaction" may not appear in the OData spec, Batch is supported in OData. Batch requests allow grouping multiple operations into a single HTTP request payload. In batch request you can define change sets. For example, Dynamics CRM has this built in. More please check batch request in OData spec.
